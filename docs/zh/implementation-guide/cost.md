# 费用预估

!!! Important "重要"

    本节中描述的成本估算只是示例，它们是根据假设计算的，可能会因您的环境而异。


您需要承担运行解决方案时使用亚马逊云科技各个服务的成本费用。 截至2022年10月，影响解决方案成本的主要因素包括：

- 要摄取的日志类型
- 要摄取/处理的日志量
- 日志消息的大小
- 日志的位置
- 额外的特性

以下示例将演示 10/100/1000 GB 每日日志摄取的成本估算。 总成本由 [**Amazon OpenSearch 费用**](#amazon-opensearch)、[**处理费用**](#_1) 和 [**额外特性费用**](#_4)。


!!! Note "信息"

    所有成本估算均基于亚马逊云科技北佛吉尼亚区域 (us-east-1) 中的 AWS 服务价格。

## 处理费用

### AWS 服务日志

摄取和处理服务日志的成本可能会根据日志位置等情况而有所不同。

### Amazon S3 中的日志

!!! note "信息"

    从 S3 提取 AWS 服务日志会产生非常低的 SQS 和 S3 请求费用，通常在 AWS 免费套餐内。

以下是假设：

- AWS 服务每 5 分钟以 gzip 格式将日志保存到 Amazon S3（大多数 AWS 服务以 gzip 格式输出日志）。
- S3 中 4MB 的压缩日志文件的原始日志大小约为 100MiB。
- 一个 1GB 内存的 Lambda 处理一个 4MiB 的日志文件大约需要 26 秒，即每 MiB 原始日志需要 260 毫秒
- 最大压缩日志文件大小为 5MiB。

您每天有 `N` GB 原始日志，每日成本估算如下：

- Lambda 成本 = 每 MiB 260 毫秒 x 1024 MiB x `N` GB/天 x 每毫秒 0.0000000167 美元
- S3 存储成本 = 每 GB 0.023 美元 x `N` GB/天 x 4%（压缩）

处理 AWS 服务日志的每月总成本为：

**每月总成本** =（Lambda 成本 + S3 存储成本）x 30 天

| 每次日志量 | 每日 Lambda 费用 ($) | 每次 S3 存储费用 ($) | 每月费用 ($) |
|-------|------------------|----------------|----------|
| 10    | 0.044            | 0.009          | 1.610    |
| 100   | 0.445            | 0.092          | 16.099   |
| 1000  | 4.446            | 0.920          | 160.986  |


### 应用日志

!!! important "重要"

    如果您有多种日志格式（索引），则需要对每种格式进行成本估算。

摄取和处理应用程序日志的成本可能会根据日志的位置和缓存[Log Buffer](./applications/index.md#log-buffer)的选择情况而有所不同。然而，非常建议您使用日志缓存。因为日志缓存不仅可以起到日志聚类的作用，还能在业务高峰期帮您的日志流量削峰以保护您的OpenSearch集群。如果您非常确定您的日志量很小或者OpenSearch集群的规模足以应对日志量的激增，那么可以不使用日志缓存。

#### 从 Amazon S3 提取日志

成本估算基于以下假设和事实：

- 平均日志消息大小为 1 KB。
- 每日日志量为`L`GB。
- Lambda 内存为 1024 MiB。
- 日志代理在达到缓冲区大小（50 MiB）或缓冲区间隔（60 秒）阈值时将日志传输到 Amazon S3。满足的条件首先触发将数据传输到 Amazon S3。
- Gzip格式的压缩率为20%。
- 处理小于 10 MiB 的日志的 Lambda 运行时为 1000 毫秒。
- 每个日志文件将遇到一个 S3 PUT 请求来上传日志文件，以及一个 S3 GET 请求来读取日志文件。
- Amazon S3 上保存的每日文件数`N`= Max（每天`L`GB x 1024 / 50 MiB，24 小时 x 60 秒/小时）
- Amazon S3 上保存的文件`V`的每日数量 = 每天`L`GB x 20% 压缩率

基于上述假设，以下是每日成本估算公式：

- Amazon S3 存储成本 = 每 GB 0.023 USD x 每日文件量`V`
- Amazon S3 请求成本 = （$每 1000 个请求 0.005 USD + 每 1000 个请求 0.0004 USD） x 每日文件数 `N` / 1000
- Lambda 成本 = 0.0000000167 USD/1ms x 1000 毫秒/每次调用 x 每日文件数`N`

**每月总成本** =（Lambda 成本 + S3 存储成本 + S3 请求成本）x 30 天

| 每日日志量 (GB) | 每日文件数 | 每日S3存储费用 ($) | 每日S3请求费用 ($) | 每日Lambda费用 ($) | 每月费用 ($) |
| --------------------- | --------------------- | ------------------------- | ------------------------- | --------------------- | ---------------- |
| 10                    | 1,440                 | 0.046                     | 0.008                     | 0.024                 | 2.335            |
| 100                   | 2,048                 | 0.460                     | 0.011                     | 0.034                 | 15.158           |

#### 通过 Amazon Kinesis Data Streams 提取日志

成本估算基于以下假设和事实：

- 平均日志消息大小为 1 KB。
- 每日日志量为`L`GB。
- Lambda 内存为 1024 MiB。
- 每个 Lambda 调用处理 1 MiB 日志。
- 一个 Lambda 调用处理 Kinesis 的一个分片，Lambda 可以扩展到更多并发创新以处理多个分片。
- 处理小于 5 MiB 的日志的 Lambda 运行时为 500 毫秒。
- 提供 30% 的额外分片来处理流量抖动。
- 一个 Kinesis 分片引入日志大小为 = 1 MiB/秒 x 3600 秒/小时 x 24 小时 x 0.7 = 60.48 GB/天。
- 所需的 Kinesis 分片数 `S` = Round_up_to_next_integer（每日日志量 `L` / 60.48）。


基于上述假设，以下是每日成本估算公式：

- Kinesis 分片小时成本 = 0.015 USD/分片小时 x 每天 24 小时 x `S`分片
- Kinesis PUT 有效负载单位成本 = 每百万单位 0.014 USD x 100 万/GB x `L`GB/天
- Lambda 成本 = 0.0000000167 USD/1ms x 500 毫秒/调用 x 1000 次调用/GB x `L`GB/天

**每月总成本** =（Kinesis Shard 小时成本 + Kinesis PUT 有效负载单位成本 + Lambda 成本）x 30 天

| 每次日志量 (GB) | 分片数 | 每日 Kinesis 分片小时费 | 每日 Kinesis PUT 有效负载费用 | 每次 Lambda 费用 | 每月总费用 ($) |

## 额外特性费用

!!! Note "信息"

     如果您选择不使用 Log Hub 控制台中的附加功能，则不会向您收费。

### 访问代理

如果通过 Log Hub 部署并创建了[访问代理](./domains/proxy.md)，将收取以下费用。

成本估算基于以下假设和事实：

- 代理模块配置了 2 个 t3.large 实例。
- 采用了 1 年全部预付预留实例定价模型。
- t3.large 实例的 1 年全部预付预留实例为 $426.612/年。
- 每台 EC2 实例的 EBS 大小 为 8 GB。

基于上述假设，这里是每月成本估算公式：

- EC2 成本 = EC2 $426.612 x 2 / 12 个月 = $71.1/月
- EBS 成本 = 0.1 GB/月 x 8 GB x 2 = $1.6/月
- 弹性负载均衡器成本 = 每 ALB 小时 $0.0225 x 720 小时/月 = $16.2/月

**每月总费用** = $71.1 EC2 成本 + $1.6 EBS 成本 + $16.2 弹性负载均衡器成本 = **$88.9**
